清皊の糯叽机正则生成器 V1.0

① API 接口与世界书 最高优先级
世界书会被自动写入 AI Prompt 的最前面。之后 AI 生成正则、HTML、字段、示例内容时，必须先阅读并遵守这里的规则。
世界书不会作为单独字段导出到糯叽机规则里，而是自动合并进 aiPrompt 顶部，确保角色触发该正则时先遵守世界书。
② 需求与基础信息
这里才是用户真正填写需求的地方：先写你想做什么、想显示成什么样、用户怎么触发。pattern 属于高级设置，默认自动生成，不需要普通用户手填。
高级正则设置：一般不用改
pattern 会根据包裹标签 tag 自动生成。只有你明确知道正则写法时才需要手动改。
会优先读取“我想生成什么正则 / 模板”和“补充要求”，再结合规则名称、分类、tag、数据格式、显示方式、触发关键词和世界书，自动选择 UI 设置、功能设置、输出结构里的标签。
③ UI 设置标签 只管长相
这一层只负责视觉：容器、布局、风格、图片、动效。不要在这里想字段和玩法。
④ 功能设置标签 只管用途
这一层负责规则要做什么：展示、互动、游戏、报告、票据、长文、工具等。
⑤ 输出结构标签 替代原来的字段手填
原来的“数据字段与 AI 输出格式”太复杂，现在改成标签：选你需要的数据结构，系统自动生成字段、示例和 Prompt。
⑥ 一键生成区 真 AI 生成
这里负责生成完整糯叽机 JSON。生成后先看下面的“生成结果”，再把其中的 HTML 一键放到预览代码框里精修。
配置 API 后会根据②需求、世界书和标签直接生成完整规则；未配置完整 API 时不会自动降级，避免生成占位符垃圾。
⑦ 生成结果
这里是最终可导入糯叽机的 JSON。需要修改界面时，点击按钮把 JSON 里的 htmlTemplate 提取到下方 HTML 代码框。
点击“AI 一键生成完整 JSON”。
生成后会自动检查：外部 CDN、复杂模板语法、触发词、关键数值是否依赖 JS、图标是否依赖 JS 后渲染等常见失效原因。
⑧ HTML 修改与实时预览 最终效果
直接在这里修改生成的 HTML。你改代码框，下面的预览窗口会自动变化；复制/下载 JSON 时也会使用这里最新的 HTML。
预览会用隐藏的示例数据渲染；如果你只改 HTML 样式/结构，预览会立即跟着变。
